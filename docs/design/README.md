# Handoff: NetDisplay 界面（主面板 + 托盘菜单）

## Overview
NetDisplay 是一个对称的「第二显示器」工具：两台电脑（Mac/Windows 任意方向）配对一次后，一端把整块屏幕或单个程序窗口投给另一端当扩展显示器。本次交付的是**统一视觉语言的完整界面设计**：主面板窗口、托盘/菜单栏右键菜单、配对弹窗、中转设置弹窗、接收显示窗口，含浅色/深色双主题。

## About the Design Files
本包中的文件是 **HTML 制作的设计参考原型**（`NetDisplay 原型 v2.dc.html`），展示预期的外观与交互，**不是可直接使用的生产代码**。任务是在目标代码库的现有环境（如 Qt、Electron、SwiftUI、WinUI 等）中**重新实现**这些设计；若尚无环境，请为跨平台桌面应用选择合适的技术栈再实现。

## Fidelity
**High-fidelity（高保真）**：颜色、字号、间距、圆角、状态措辞均为最终意图，请按像素还原（用目标平台的等价控件）。

## 核心产品原则（必须贯彻）

1. **连接方式对用户隐形**：绝不出现「直连/中转/自动」三选一。程序自行探测走直连还是中转；界面只把结果作为状态展示，如 `已连接 · 直连 · 3ms` / `已连接 · 中转 · 310ms`。「强制走中转」只在中转设置里作为兜底开关。
2. **文案用用户的话**：配对、连接、投射、接收、待命、等待连接中。禁止 listen/dial/socket/端口/A位B位 等实现词汇。
3. **投射与接收互斥**：同一对设备同一时刻只有一个来源。投射中→接收服务不可用（置灰）；接收中→投射不可用。切换时短暂显示「切换中…」（约 0.9s）。
4. **两端统一视觉**：Mac 是菜单栏 App、Windows 是窗口面板，但图标、配色、状态措辞、信息分层完全一致。

### 2. 托盘/菜单栏右键菜单（246px 宽，5px 内边距，条目 6px 圆角，hover panel2 底）
Mac 顶栏图标与 Windows 托盘图标共用同一菜单。分四节，1px line 分隔：

**第一节 — 只有两项：投射 / 接收服务**
- 「投射」（accent 投射图标）：空闲时右侧「开始 ▸」（accent）；投射中标题变「投射中 · {内容名}」+ 副行「目标：{设备名} · 直连 · 3ms」+ 右侧「断开」（err 红）；接收中整项 45% 透明度、右侧「接收中不可用」。
- 「接收服务」（recv 接收图标）：关闭时「启动接收服务」；等待中「接收服务 · 等待连接」+ 右侧「关闭」；接收中「接收投屏中」+ 副行「来源：{设备名} · 直连 · 3ms」+ 右侧「断开」（断开投屏不关服务）；投射中置灰「投射中不可用」。

**第二节 — 投射内容（仅当所选配对设备已连接且未在接收时显示）**
- 小标题「投射内容 · 最近窗口」。整块屏幕 + **最近使用的最多 8 个程序窗口**直接列出（✓ 标记记忆的选择，选择被持久记住）；再多的放「更多窗口 ▸」二级菜单（向屏幕内侧弹出，即右缘托盘向左弹）。

**第三节 — 已配对设备（内容随接收服务状态变化）**
- 未开接收服务（=投射模式）：标题「投射目标 · 已配对设备」，行 = ◉/○ 选择 + 圆点 + 名称 + 状态 + **连接/断开**操作（断开 err 红、连接 accent）。
- 已开接收服务：标题「等待接收 · 已配对设备」（接收中为「接收中 · 已配对设备」），只显示连接状态，无连接操作。
- 一次只能连一台：连接新设备自动断开其他。投射/接收进行中设备不可切换（非当前设备 45% 透明，点击 toast 提示）。
- 「＋ 添加设备…」**仅在配对设备列表为空时显示**。

**第四节 — 显示（作为显示器时）**
- 「分辨率」「缩放」两行，选项为小 chip（10px，选中 recv 边框+recv-weak 底）直接展示。
- 「更多画质… {当前预设} ▸」二级菜单：流畅优先（30fps·低码率）/ 平衡（60fps·自动）/ 清晰优先（60fps·20Mbps）三档预设（✓ 标记，联动主面板帧率/码率）+「帧率 / 码率（主面板）…」跳主面板画质设置。

**尾部**：中转设置…（弹出小窗，见 4）、打开主面板、退出。

## Interactions & Behavior（状态机）

**角色 role**：`standby | switching | casting | receiving`
- standby →（点开始投射，需已选设备且在线）→ switching(0.9s) → casting
- standby →（服务等待中 + 对方开始投）→ switching → receiving（自动，无需操作）
- casting/receiving →（停止/断开）→ standby；接收断开后服务保持 waiting
- 断开所选设备连接 → 强制回 standby

**接收服务 recvSvc**：`off | waiting`。默认 off。开启→waiting；waiting 点关闭→off；接收中点按钮→只断开投屏（服务仍 waiting）。

**连接 conn（每设备）**：`off | connecting(1.2s) | on`。全局互斥（连一台自动断其他）。开始投射时隐式连接。

**投射内容选择 pickSel**：默认整块屏幕；**持久记忆**（重启后保留），主面板与托盘共用同一份记忆。

**Toast**（屏幕底部居中，text 色底 panel 色字，2.6s 自动消失）用于所有反馈：连接成功/断开、配对成功、对方离线、投射中不可用、画质已设为X 等。全部用人话。

**错误提示示例**：「对方离线，无法连接」「配对码错误，请核对后重试」「正在投射，接收服务不可用」。

## State Management
- `theme`（跟随系统，可手动切换）、`role`、`recvSvc`、`tab`、`selectedId`、`pickSel`（持久化）、`devices[{id,name,online,conn}]`（持久化：id/name/配对关系）、`quality{res,scale,fps,rate}`（持久化）、`trayQuality` 预设名、`relayAddr/relayToken/forceRelay`（持久化）、`localName`（持久化）。
- 数据获取：设备在线状态、连接质量（直连/中转 + 延迟）需实时探测并推送到 UI；窗口列表按最近使用排序实时获取。
