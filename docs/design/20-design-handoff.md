# NetDisplay 界面重设计 · 需求交接（给设计师）

> 目的：把 Mac 端和 Windows 端的界面**重新设计、统一风格**。
> 本文只讲**产品要做什么、界面上要承载哪些信息和操作**，不给具体布局/控件方案——那是设计要产出的。

---

## 一、这个产品是什么

**把一台电脑的画面，投到另一台电脑上当扩展显示器用。**

- **对称、任意方向**：Mac→Windows、Windows→Mac、Win→Win、Mac→Mac 都行。每台机器**既能把自己投出去，也能接收别人投过来**——不是「主机/从机」，是对等的两台。
- **两种投射内容**：① 整块屏幕（当第二显示器）；② **单个程序窗口**（只把某个 App 的窗口投到对面，是一等功能，不是附属）。
- **主要场景是局域网**（同一个 Wi-Fi / 网线 / 雷雳网桥），延迟低、画质高；跨互联网也能用（走中转服务器），延迟会高一些。

---

## 二、最重要的设计原则（务必贯彻）

### 1. 连接方式对用户**隐形**——不要让用户选「直连 / 中转」
这是本次重做的**核心动机**。旧版让用户选「直连还是中转」，用户看不懂、骂得很凶——因为这是个他根本答不上的问题（「我和对方在不在同一个网？」他不知道）。

**参照 TeamViewer / AnyDesk / Parsec 的模型：**
- 用户**只做一件事：交换一个配对码**。
- 是走「同网直连（快）」还是「中转服务器（慢）」，**由程序自己探测决定**，用户不选、不填 IP。
- 界面只把结果**显示**出来当信息看，例如：`已连接 · 直连 · 3ms` / `已连接 · 中转 · 310ms`。这是**状态展示，不是让用户点的选项**。
- 「强制走中转」这类兜底开关，放进**高级设置**里，默认不出现。

> 一句话：**transport 是程序算出来的状态，不是用户要做的选择。** 设计上不要出现「直连/中转/自动」这种三选一。

### 2. 两端**统一视觉语言**
Mac 端形态是**菜单栏 App**（轻、常驻顶栏），Windows 端形态是**窗口面板**——形态可以不同，但**图标、配色、状态措辞、文案风格、信息层级必须一致**，让人一看就知道是同一个产品。

### 3. 文案用**用户的话**，不暴露实现细节
不要出现 listen / dial / register / join / A位 / B位 / socket / 端口 这类词。用户看到的是「配对」「连接」「投射」「接收」「待命」这种日常语言。

---

## 三、界面需要承载的内容与操作（按功能列，不含布局）

### A. 配对（首次把两台设备关联起来，一次性）
- 一端**显示一个配对码**；另一端**输入这个码**。配对成功后**持久记住**，以后开机自动连、不用再输码。
- 能看到**已配对设备的列表**，每个能**解除配对 / 重命名**。
- 需要「本机名称」（让对方认得出是哪台机器）。

### B. 连接状态（配对之后，常驻显示）
- 当前是否连上：`未连接 / 连接中… / 已连接`。
- 连上后显示**这条连接的性质与质量**（纯信息）：直连还是中转、延迟大概多少（如 `直连 · 3ms`）。
- 一个**断开**入口。

### C. 角色开关（本机此刻在干嘛）——这是主界面的核心
本机有三种状态，用户主要就在这里操作：
- **待命**：空闲。屏幕/窗口显示「等待对方投射…」。对方一旦开始投过来，**自动切到「接收显示」**，不用手动。
- **投射本机（我是来源）**：把本机画面投给对方。需要能选**投什么**：
  - 整块屏幕，或
  - 某一个程序窗口（给一个可选的窗口/App 列表）。
- **接收显示（我是目标）**：显示对方投来的画面（通常从「待命」自动进入）。

外加一个**停止 / 回到待命**的操作。
> 注意：同一对设备**同一时刻只有一个来源**（不能两台互投）。切换「谁投谁」时界面可能要短暂显示「切换中…」。

### D. 画质设置（当本机**被别人当显示器**时生效）
- 分辨率、缩放（HiDPI，字别太小）、帧率、码率。
- 归类清楚：这是「**作为显示器时**的画质」，**别和「投射」混在一起**。可以放二级/折叠区。

### E. 高级设置（默认折叠，普通用户不用碰）
- 中转服务器地址 + 访问 token（可以内置一个默认值，用户一般不改）。
- 「强制走中转」开关（企业网禁 P2P 时的兜底）。

### F. 反馈与提示
- 连接失败、对方离线、配对码错误等，要有**清楚的人话提示**。
- 投射/接收进行中，要有明确的「正在投射给 XXX / 正在接收 XXX 的画面」的标识。

## 四、两端形态提示（供设计时拿捏差异）

| | Mac | Windows |
|---|---|---|
| 形态 | 菜单栏 App（顶栏图标 + 下拉菜单/小面板） | 桌面窗口面板 |
| 常驻 | 常驻顶栏，点开才展开 | 可最小化到托盘 |
| 接收画面 | 独立窗口显示 | 独立窗口显示 |
| 共同点 | **同一套图标/配色/文案/状态措辞/信息分层** | 同左 |

Mac 受限于菜单栏空间，信息要更精炼；Windows 面板可以铺得开一点。但**两端是同一个设计系统的两种落地**，不是两个产品。

---

## 五、一句话总结给设计师

> 做一个**对称的「第二显示器」工具**：两台电脑配一次对，之后一端点「投射本机（整屏或某个窗口）」，另一端自动显示。**连接怎么走（直连/中转）由程序决定、只做展示**，用户永远只面对「配对码 + 一个投射开关 + 画质」。Mac 是菜单栏、Windows 是面板，但**风格统一**。
