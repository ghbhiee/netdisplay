# NetDisplay 交互模型（两端统一 UX 的唯一依据 / SoT）

> 像 `02-protocol.md` 是线上协议的 SoT 一样，本文是**两端 UI/交互**的 SoT。
> 两端必须按同一个模型做，只是控件形态不同（Mac 菜单栏 / Windows 面板）。

## 一、现状为什么乱（要改掉的）

- 两端都把「发送」和「接收」当成**两条独立流程**，每条又各带一套 `直连/中转` 选择 → 用户在多处重复设连接方式，而且「谁 listen 谁 dial / 谁 register 谁 join」这种实现细节**暴露给了用户**。
- Windows：顶部「连接模式」配的是**接收**，下面「发送端」又有**并列的**「▶ 直连发送 (:47800)」「☁ 中转发送」两个按钮 —— 发送的连接方式和接收的连接方式各设一套。
- Mac：菜单里 `开始投送` / `接收投射` / `模式(直连/中转)` / `中转设置` 散落各处；而且**接收只实现了中转，不能直连**。
- 结果：搞不清谁是目标、谁是来源、谁和谁配对、谁连谁。

## 二、统一模型：**一次配对 → 常驻连接 → 随时切换谁投谁**

三个概念清晰分层（正好对应协议 v1.4「连接与投射解耦」，所以**协议不用改，这是纯 UI 层**）：

### 1) 配对（Pairing）——一次性，两端填一样
一对设备互相认识一次，之后持久免码。
- **中转配对**：中转服务器地址 + token（两端填同一个）+ 首次一个配对码。
- **直连配对**：对方地址（局域网 IP / USB4 网桥）。
- 状态：`未配对` / `已配对 ✓`（可解除配对）。

### 2) 连接方式（Transport）——配对层的**一个**设置，收发共用
`[自动] / 直连（同网/USB4，低延迟）/ 中转（跨网络）`。**只设一次，发送和接收都用它**，不再各设一遍。
- 自动：有可达的直连地址就直连，否则回退中转。

### 3) 角色开关（Role）——**一个开关，随时切**，用户不用想「谁连谁」
本机三态：
- **待命**：空闲，画面显示「等待投射…」；对方一旦开始投射，**自动切到「接收显示」**。
- **投射出去（来源）**：选投射源（整屏 / 某个窗口），把本机画面投给对方。
- **接收显示（目标）**：显示对方投来的画面（通常由"待命"自动进入，无需手动）。

一个「停止 / 弹回」按钮回到待命。

**规则**：
- 配对后**连接常驻**；投射是连接之上的开关——切换投射源、开/关投射都**不断连**。
- 同一对里**同一时刻只有一个来源**。你点「投射出去」，对方自动成为目标；对方想投，先停你的。
- **连接编排由 app 自动处理**，用户看不到：中转=来源 `register`、目标 `join`；直连=来源 `listen(47800)`、目标 `dial`。**角色 + 连接方式**决定谁 listen/dial、谁 register/join。

### 画质设置的归属
分辨率 / 缩放 / 帧率 是**目标端（接收方）**请求的（HELLO.screen），**目标端设、来源端采纳**；码率来源端可覆盖。UI 上归到「作为目标时的画质」，**不要和"发送"混在一起**。

## 三、两端 UI 落地（同一套，控件形态不同）

**一个「连接设置」面板（两端字段一致）**：
`连接方式[自动/直连/中转]` →（直连填`对方地址`；中转填`服务器+token`）；`配对状态/配对码`；`作为目标时的画质[分辨率/缩放/帧率/码率]`。
一个 `[连接] / [断开]`。

**主控制（连接后）**：
`[● 投射本机 ▾（源：整屏 / 某窗口）]  [■ 停止/弹回]`。接收是自动的（对方投来即显示）。

**各端具体**：
- **Mac 菜单栏**：`连接设置…`（面板）→ `连接`/`断开`；连上后 `投射本机 ▸（源子菜单）`、`停止投射`；画质子菜单标注「（作为目标时）」。去掉散落的 `模式`/`中转设置`/独立 `接收投射` 入口——统一到「连接设置 + 角色开关」。**补：接收支持直连**（目标 dial 对方 :47800）。
- **Windows 面板**：一个设置面板（连接方式 + 直连/中转 + 画质）+ 一个 `连接`；主界面连上后就是 `投射本机 ▾ / 停止`。**去掉「▶ 直连发送 (:47800)」「☁ 中转发送」两个并列按钮**——合并成一个「投射本机」，连接方式由设置决定，不在按钮上区分。

## 四、迁移要点
- **直连接收两端都要支持**（目标 dial 对方 :47800）。Mac GUI 目前只做了中转接收，补直连。
- CLI 入口（`--host` / `--server` / `--secret`）**保持不变**，不受 UI 重构影响。
- 协议 **不改**（v1.4 已解耦连接与投射）。
